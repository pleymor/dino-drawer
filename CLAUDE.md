# Dino Drawer — project notes for Claude

Pipeline that produces a single scientific dinosaur photo-illustration per species, optimises it, and publishes to Cloudflare R2 with a thumbnail. Output consumed by a static site via `published/catalog.json`.

## Pipeline (6 steps)

`papers → images → vision → synthesis → diffusion → compose → publish`

| Step | Module | Input | Output | Cost |
|------|--------|-------|--------|------|
| papers | `research.papers` | species name | `out/<slug>/papers.json` (Wikipedia full article + papers) | free (arXiv + Wikipedia API) |
| images | `research.images` | species name | `out/<slug>/refs_raw/` + `refs_raw.json` | free (Wikimedia) |
| vision | `vision.classifier` | raw images | `out/<slug>/refs.json` (top-N **body** refs only) | Gemini VLM (parallel, ~4 concurrent) |
| synthesis | `synthesis` | papers + body refs | `out/<slug>/factsheet.json` | 1 Gemini text call |
| diffusion | `image` | factsheet | `out/<slug>/hero.png` | 1 Gemini image call |
| compose | `compose` | `hero.png` | `out/<slug>/final.png` (PIL resize + black canvas) | free |
| publish | `publish` | `final.png` + factsheet | R2 upload (full + thumbnail) + `published/catalog.json` | free (R2 bandwidth) |

Everything is Gemini-backed. The `synthesis/ollama_client.py` module name is legacy — it's Gemini under the hood.

## FactSheet: mandatory blocks contract

The factsheet is the source of truth for everything downstream (image prompt + catalog content). It enforces a "minimum set of mandatory information" contract: each block has its own `source_ids` and **every field must be sourced or marked `"no data in corpus"`**. The LLM is forbidden from inventing numbers.

| Block | Fields | Role |
|------|--------|------|
| `dimensions` | `body_length, hip_height, skull_length, forelimb_length, tail_length, body_mass` (all free text) | Drives proportions / scale cues in the image prompt |
| `integument` | `integument_type, coloration, keratinous_structures, ontogenetic_variation` | Drives skin/feathers/colour cues |
| `posture` | `stance, typical_posture, locomotion_mode` | Reinforces the horizontal-theropod stance |
| `habitat` | `geological_period, biome, region_or_formation` | Drives the background of the image |
| `signature_traits` | `text` (free-form catch-all) | Visually distinctive features that don't fit elsewhere |

Each block has `source_ids: list[str]` (one entry per cited `Reference.id`). The validator in `models.py` rejects unknown source ids. The text values themselves should also contain inline citations like `"~12.3 m (Sue) [paper:9][wikipedia]"`.

## Where prompts live

The hero illustration prompt is composed at generation time from two parts:

- **Common (universal) prompt** — `src/dino_drawer/image/diffusion.py::_COMMON_HERO_PROMPT`. Photographic style, framing, pose (head-on charge), theropod horizontal posture, theropod skull anatomy (Sue/Stan-grade), camera. Same for every species. Positive framing only (no `NOT X` negations — image gen amplifies mentioned concepts).
- **Species-specific prompt** — `factsheet.image_prompt`, generated by the LLM per the contract in `src/dino_drawer/synthesis/prompts.py::_SCHEMA_HINT`. Should cover ONLY: paleoenvironment (from `habitat`), tegument-by-mass (from `integument`), colour palette (from `integument.coloration`), unique anatomical features (from `signature_traits` + `dimensions`), body length (from `dimensions.body_length`).
- **Composer** — `_hero_prompt(factsheet)` in `diffusion.py`: prepends common, appends the no-text footer.

If you edit the common prompt or `_SCHEMA_HINT`, see iteration loops below.

## Common iteration loops

| Goal | Command | Cost |
|------|---------|------|
| First-time generation for a new species | `make run SPECIES='Triceratops horridus'` | full pipeline |
| Re-do everything from synthesis (you edited the synthesis prompt or factsheet rules) | `make run SPECIES='X' ARGS='--force-step synthesis'` | 1 synth + 1 image call |
| Re-do image only (you edited `_COMMON_HERO_PROMPT` or `factsheet.image_prompt` by hand) | `make regen SPECIES='X'` | 1 image call + compose + publish |
| Apply prompt change to every published species | `make regen-all` | n × (image + compose + publish) |
| Publish a species (after `regen` already ran image+compose) | `make publish SPECIES='X'` | free |
| Remove a species from R2 + catalog | `make unpublish SPECIES='X'` | free |

## Gotchas

- **`out/` is gitignored.** Fresh clones have no prior artefacts. You cannot run `synthesis` or `image` standalone on a fresh clone — only `make run` works because it does papers + images first. Don't promise the user a "quick re-synthesis" without checking `out/<slug>/papers.json` exists.
- **`venv` is not committed.** Check `.venv/bin/python` exists before running anything; `make install` creates it via `uv`.
- **`google-api-core` is a real dep** (used in `clients/gemini.py` for exception types). It's listed in `pyproject.toml` because it's no longer pulled transitively by `google-genai>=2.0`.
- **Wikipedia is fetched as the full article** via `action=query&prop=extracts&explaintext=1` (not `/page/summary`) — the synthesis LLM gets up to `_WIKI_CHAR_LIMIT` (12 k) chars including Description/Anatomy/Size sections.
- **Required env vars** (in `.env`): `GEMINI_API_KEY`, `R2_BUCKET`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_ENDPOINT`, `R2_PUBLIC_BASE_URL`. `HF_TOKEN` is unused (legacy).
- **Thumbnails**: `publish` uploads both `<slug>.webp` (full) and `<slug>_thumbnail.webp` (max 400 px long edge, quality 80). The catalog entry contains both `image_url` and `thumbnail_url`.
- **Catalog is committed locally** at `published/catalog.json`, NOT uploaded to R2. The static site reads it from the repo. Entries pre-dating the mandatory-blocks refactor have the old `annotations / skull_view / size` fields purged but not yet backfilled with the new `dimensions / integument / posture / habitat / signature_traits` blocks — re-running `make run --force-step synthesis` + `make publish` will repopulate them.

## Layout

```
src/dino_drawer/
  __main__.py        CLI (orchestrates agent.run)
  agent.py           DinoDrawerAgent — runs the 6 steps with skip-if-exists logic
  models.py          pydantic models (FactSheet with mandatory blocks)
  clients/gemini.py  Gemini SDK wrapper with retry + exception handling
  research/          papers + images scrapers (Wikipedia full article)
  vision/            Gemini VLM-based body-ref classification + description
  synthesis/         prompts.py (LLM input with mandatory-block schema) + ollama_client.py + main.py
  image/             diffusion.py — single hero gen, no skull, no silhouette
  compose/           render.py::screenshot — PIL resize + black canvas → final.png
  publish/           r2.py (boto3 S3-compat) + manifest.py (catalog) + __main__.py
```

## Conventions

- `--force-step <step>` re-runs that step and everything after (skip-if-exists is bypassed).
- Source IDs in `factsheet.json` are sanitized: unknown IDs from LLM hallucination become `"wikipedia"` (or the first known id) rather than failing validation. See `synthesis/main.py::_sanitize_source_ids`.
- Each species' `out/<slug>/factsheet.json` is the source of truth for what goes into the catalog. Editing it directly before `make publish` is a valid escape hatch.
- When the LLM has no data for a mandatory-block field, it must write the literal string `"no data in corpus"` rather than making something up. An entire block with no source backing can have `source_ids: []` provided all its fields say `"no data in corpus"`.
